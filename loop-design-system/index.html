Brand Identity

로고

Loop의 공식 로고입니다. L + ∞ + p 구조로, 순환 대여 서비스의 무한 루프 컨셉을 시각적으로 표현합니다.

Primary Logo (Gradient)

색상

#38BDF8 → #0EA5E9

구조

L (올 라운드) + ∞ (인피니티 교차) + p (커스텀 볼)

최소 크기

120px (디지털) / 30mm (인쇄)

White Version

다크 배경, Footer, 히어로 섹션용

Flat Version

단색 사용이 필요한 경우 (#00BCFF)

Dark Background + Gradient

다크 배경에서도 그라데이션 로고 사용 가능 (대비율 7.5:1, WCAG AAA)

사용 규칙

✅ Do

  • • 충분한 여백 확보 (최소 L 세로획 폭만큼)
  • • 공식 색상만 사용 (그라데이션 / #00BCFF / 화이트)
  • • 비율 고정 (가로세로 비율 변경 금지)
  • • 밝은 배경 → 그라데이션/플랫
  • • 다크 배경 (#101828) → 그라데이션 또는 화이트
  • • 복잡한 배경/사진 → 화이트
  • • 단색 제한 → #00BCFF

❌ Don't

  • • 로고 색상 임의 변경
  • • 로고 위에 그림자/효과 추가
  • • 로고 회전/기울이기
  • • 120px 미만 크기로 사용

Usage Guide

사용 가이드

Loop 디자인 시스템을 일관되게 적용하기 위한 가이드라인입니다.

색상 위계

  • Primary (Sky Blue 400) — CTA 버튼, 핵심 액션, 브랜드 포인트
  • Secondary (Violet 400) — 서브 액션, 보조 인터랙션
  • Success (Emerald) — 완료, 성공 피드백
  • Warning (Amber) — 주의, 기한 알림
  • Accent (Yellow Gold) — 리뷰, 별점, 프로모션

그라데이션 활용

  • 01히어로 영역에만 3-color 그라데이션 사용 (sky→violet→rose)
  • 02CTA/배너에는 2-color 그라데이션 (sky→violet)
  • 03추천/완료 배경에는 50 단계의 은은한 그라데이션
  • 04본문 영역에는 그라데이션 사용 자제 — 가독성 우선

대비율 가이드

  • ✓버튼 텍스트 → 흰색 on sky-400: WCAG AA 충족
  • ✓본문 텍스트 (#4A5565) on 배경 (#F9FAFB): 7:1 이상
  • ✓뱃지 텍스트는 400~600 단계 색상 사용으로 가독성 확보
  • ⚠밝은 배경(50~100)에는 400 이하 텍스트 사용 금지

Z세대 타겟팅

  • 💡파스텔 톤 기반으로 부드럽고 친근한 브랜드 이미지 구축
  • 🎨그라데이션으로 역동적이고 트렌디한 느낌 표현
  • ✨높은 채도 + 밝은 톤으로 에너지와 즐거움 전달
  • 📱모바일 퍼스트 — 모든 컴포넌트는 44px 이상 터치 타겟

Visual Theme

비주얼 테마 & 분위기

Loop.day의 디자인 철학 — "밝고 가벼운 신뢰". 렌탈이라는 금전 거래를 파스텔 톤으로 부드럽게 감싸면서도, 비교 데이터의 정확성과 투명성을 해치지 않는 균형.

브랜드 철학

페이지는 밝은 off-white 캔버스(#F9FAFB)에서 시작한다. 순수 white가 아닌 미세하게 따뜻한 회색이 배경이 되어, 그 위의 white 카드들이 자연스럽게 떠오르는 느낌을 준다. 이 1단계 elevation이 Loop의 기본 공간감이다.

브랜드 아이덴티티는 Sky Blue(#00BCFF) → Violet(#A684FF) 그라데이션으로 압축된다. 타이포그래피는 Inter — 가장 범용적이고 가독성 높은 sans-serif. 콘텐츠(렌탈 정보, 가격, 비교 데이터)가 주인공이고, UI는 무대.

Key Characteristics

  • •밝은 off-white(#F9FAFB) 캔버스 위 white 카드 — 1단계 자연 elevation
  • •Sky Blue #00BCFF primary → Violet #A684FF secondary 축
  • •3-color 히어로 그라데이션(#74D4FF → #C4B4FF → #FFA1AD) — 히어로에만
  • •Inter font, weight 400–700, display 최대 36px
  • •10px border-radius가 기본 — 부드럽지만 pill 아님
  • •가벼운 2-layer shadow (rgba(0,0,0,0.1) 기반) — Tailwind 표준 shadow 체계
  • •Heading은 #101828 (warm near-black), body는 #4A5565 (slate)
  • •모바일 퍼스트 375px — Z세대 = 모바일 네이티브

Do's & Don'ts

디자인 규칙

Loop 디자인 시스템의 핵심 규칙. 일관성을 유지하기 위해 반드시 지켜야 할 Do와 절대 하지 말아야 할 Don't.

✅ Do

  • ✅Sky Blue #00BCFF를 primary CTA에만 사용 — 한 화면에 sky-400 버튼은 1~2개
  • ✅그라데이션은 용도별 구분 엄수 — 3-color(히어로만), 2-color CTA(sky→violet), 50단계(추천/완료 배경)
  • ✅카드에는 항상 border + shadow-sm — 1px solid #E5E7EB + shadow 둘 다 적용
  • ✅상태 색상은 semantic에만 — Emerald=성공, Amber=주의, Rose=에러/취소
  • ✅body text는 #4A5565 — heading만 #101828
  • ✅border-radius 10px이 기본 — 버튼, 카드, 모달, 토스트 모두 10px. 8px은 input에만
  • ✅focus ring은 sky-400 기반 — box-shadow: 0 0 0 3px rgba(0, 188, 255, 0.3)
  • ✅뱃지/태그는 100단계 배경 + 500단계 텍스트 — sky-100 bg + sky-500 text
  • ✅Inter weight 사용 규칙 — 400(body), 500(button/nav), 600(heading 2~3), 700(heading 1, display)
  • ✅모바일 375px에서 먼저 설계 — 데스크톱은 확장, 모바일이 기본

❌ Don't

  • ❌다크 모드 색상을 light 모드에 섞지 마라 — #1E2939는 footer에만
  • ❌Violet을 단독 CTA로 쓰지 마라 — Violet은 secondary. primary CTA는 항상 Sky Blue
  • ❌Rose/Pink를 장식적으로 남발하지 마라 — Rose-400은 경고/취소/에러에만
  • ❌shadow-lg 이상을 카드에 쓰지 마라 — 카드는 shadow-sm. shadow-lg는 모달에만
  • ❌pill shape(9999px) 버튼을 만들지 마라 — Loop 버튼은 10px radius
  • ❌heading 크기를 36px 이상으로 올리지 마라 — display 최대 36px
  • ❌그라데이션 텍스트를 사용하지 마라 — 그라데이션은 배경/버튼에만
  • ❌border 없이 카드를 만들지 마라 — #E5E7EB border 필수
  • ❌Yellow/Gold를 경고에 쓰지 마라 — Yellow는 리뷰/별점 전용. 경고는 Amber
  • ❌font-weight 300(light)을 사용하지 마라 — Loop는 400이 최소

Iteration Guide

  1. 배경은 항상 #F9FAFB — 카드/모달만 #ffffff. 페이지 배경에 흰색 사용 ❌
  2. Sky Blue #00BCFF가 유일한 primary interactive 색상 — 모든 CTA, focus, active는 이 색에서 파생
  3. border-radius 10px 기본 — 8px는 input에만, 4px는 태그에만, 9999px는 pill 뱃지에만
  4. shadow는 Tailwind 표준 — shadow(카드), shadow-md(드롭다운), shadow-lg(모달), shadow-xl(토스트)
  5. heading #101828, body #4A5565, secondary #6A7282 — 3단계 텍스트 위계 엄수
  6. Inter font만 사용 — weight 400/500/600/700. 300이나 800 사용 ❌
  7. 그라데이션 3종만 사용 — hero(3색/135deg), cta(2색/90deg), subtle(50단계)
  8. semantic 색상 매핑 고정 — Emerald=성공, Amber=주의, Rose=에러, Yellow=리뷰
  9. 모바일 375px에서 항상 먼저 확인 — 모든 컴포넌트는 375px에서 깨지지 않아야
  10. hover는 500 단계로 — sky-400 → sky-500, violet-400 → violet-500
